# Implementation Plan: Result Set Control, TTL/Write Metadata, Uniform Document Size, and Provider Diagnostics

**Branch**: `users/allekim/feature/issue_25` | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/users/allekim/feature/issue_25/spec.md`  
**Prior work**: Phases 1–13 (T001–T135) complete. This plan covers new spec requirements added in commit b54b962.

## Summary

The Hyperscale DB SDK spec was updated with customer gap analysis refinements adding:
1. **Provider Constants Centralization completion** — `SpannerConstants`, `OperationNamesTest`, Spanner structured diagnostics (FR-049–051)
2. **User Story 5 (P1): Result Set Control** — Top N (`limit`) and ORDER BY on `QueryRequest`; capability-gated ORDER BY for Cosmos/Spanner only (FR-052–055)
3. **User Story 6 (P2): Document TTL and Write Metadata** — per-document TTL on writes; `DocumentMetadata` (remaining TTL + write timestamp) on reads; new `DocumentResult` return type (FR-056–059)
4. **User Story 7 (P2): Uniform Document Size and Quota Limits** — 400 KB enforcement in `DefaultHyperscaleDbClient` before any provider I/O; `getDocumentSizeLimit()` API (FR-060–064)

Technical approach: extend existing Java 17 multi-module Maven project. All changes are additive or backward-compatible except `read()` return type (`ObjectNode` → `DocumentResult`), which is handled via a `.document()` accessor.

## Technical Context

**Language/Version**: Java 17 (LTS)  
**Primary Dependencies**: Jackson `JsonNode`, SLF4J, Azure Cosmos SDK v4, AWS SDK v2, Google Cloud Spanner Java client  
**Storage**: N/A (SDK, not a data store)  
**Testing**: JUnit 5 + Mockito; `maven-surefire-plugin` with ByteBuddy experimental flag for Cosmos/Dynamo  
**Target Platform**: JVM library (Maven artifact)  
**Project Type**: Single multi-module Maven project  
**Performance Goals**: Document size validation must be <1 ms; no additional network calls for validation  
**Constraints**: Backward-compatible API where possible; `read()` return type change requires `.document()` migration  
**Scale/Scope**: 3 provider adapters, 4 new types, ~6 modified classes, ~10 new/modified tests

## Constitution Check

**Portability-First Default** ✅ — ORDER BY and TTL are capability-gated; fail-fast errors preserve the portable contract  
**Thin Wrapper** ✅ — TTL, ORDER BY, and LIMIT are delegated to native provider mechanisms  
**Capability-Based API** ✅ — `ROW_LEVEL_TTL`, `WRITE_TIMESTAMP`, `RESULT_LIMIT` capability constants added; unsupported features fail fast  
**Consistent Surface** ✅ — `DocumentResult` wraps both document and optional metadata uniformly  
**Configuration-Only Portability** ✅ — no provider conditionals in application code; TTL and metadata retrieval are per-request options  
**Explicit Reliability Controls** ✅ — document size validation prevents oversized payloads reaching the network  
**Diagnostics by Default** ✅ — Spanner structured DEBUG diagnostics complete the FR-051 requirement  
**No violations requiring justification.**
